Mediantem — Proyecto Académico

Stack Tecnológico

Arquitectura full stack propuesta para el desarrollo del e-commerce, foro de comunidad y panel de administración.

Capas del sistema

Frontend

Capa de presentación
🚀

Astro 4

Framework principal. Genera HTML estático con hidratación selectiva (islas React). SEO perfecto sin JavaScript innecesario.

v4.x · SSR / SSG hybrid
⚛️

React 18

Componentes interactivos: carrito, countdown, carruseles, formularios de auth. Se monta solo cuando es necesario.

v18.x · client:visible / client:load
🎨

Tailwind CSS 3

Sistema de estilos utility-first con la paleta de Mediantem definida en tailwind.config.mjs. Responsive mobile-first.

v3.x · JIT compiler
🐻

Zustand 4

Estado global del cliente. Tres stores: cartStore, authStore, wishlistStore. Sin boilerplate.

v4.x · Estado en memoria
🔷

TypeScript 5

Tipado estático en todo el frontend. Los tipos de Product, Order, User se definen en types/index.ts como fuente de verdad.

v5.x · strict mode
📦

Vite

Bundler de Astro. Build ultrarrápido con HMR en desarrollo. Genera assets optimizados para producción.

Incluido con Astro
↓

Backend

API REST · Puerto 3001
🟢

Node.js 20 LTS

Entorno de ejecución. Permite compartir lógica de validación entre frontend y backend al usar el mismo lenguaje.

v20.x LTS
🚂

Express 4

Framework HTTP. Define los routers /api/auth, /api/products, /api/orders, /api/reviews, /api/forum, /api/stats.

v4.x · REST API
🔐

JWT + bcrypt

Autenticación sin sesiones en servidor. Access token en memoria (15 min), refresh token en cookie HttpOnly (30 días).

jsonwebtoken v9 · bcrypt v5
✅

Zod

Validación de esquemas en todos los endpoints. Si el body no cumple el schema, responde 400 con errores detallados.

v3.x · Schema validation
🛡️

Helmet + CORS

Helmet agrega headers de seguridad HTTP. CORS controla qué orígenes pueden consumir la API (solo el frontend URL).

Middlewares de seguridad
📝

Morgan

Logger HTTP de peticiones en desarrollo. Muestra método, path, status y tiempo de respuesta de cada request.

v1.x · Dev logging
↓

Base de datos

SQLite · mediantem.db
🗄️

SQLite (better-sqlite3)

Base de datos relacional embebida. Sin servidor externo, el archivo .db vive en el directorio del backend. Ideal para fase inicial.

better-sqlite3 v9.x · Síncrono
🗂️

Tablas principales

users · products · product_variants · product_images · orders · order_items · reviews · forum_posts · wishlist · refresh_tokens

10 tablas · Migraciones SQL
🔄

Transacciones

La creación de pedidos usa transacciones SQLite: inserta order + order_items + descuenta stock en una sola operación atómica.

ACID · better-sqlite3 transactions
↓

Servicios externos

APIs de terceros
☁️

Cloudinary

CDN de imágenes de producto. Optimización automática (WebP, resize, quality) y URLs de entrega global con baja latencia.

Upload vía API · URLs públicas
📧

Resend

Emails transaccionales: confirmación de pedido, recuperación de contraseña. API simple, entregabilidad alta.

SMTP moderno · Templates HTML
💳

Culqi / Niubiz

Pasarela de pago para Perú. Soporta tarjetas de crédito/débito, Yape y Plin. Se integra con un widget JS en el checkout.

Perú · Tarjetas · Yape · Plin

Patrones de arquitectura

Frontend — Streaming Architecture

Page       (Astro · SSR/SSG)
│  Hace fetch de datos en servidor
│  Genera HTML estático
└── Section  (Astro · bloque estático)
     │  Sin estado, sin JS
     └── Component (React · isla)
          │  Con estado o eventos
          └── Store (Zustand)
               └── lib/api.ts → API

Backend — MVC

Route
└── Middleware
     │  auth.js (verifica JWT)
     │  isAdmin.js (verifica rol)
     │  validate.js (Zod schema)
     └── Controller
          │  Lógica de negocio
          │  Nunca escribe SQL
          ├── Model ← queries SQL
          └── Service ← APIs externas
